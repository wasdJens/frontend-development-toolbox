# Getting Started

## Installation
- NodeJS ist für alle gängigen Betriebssysteme Verfügbar
- https://nodejs.org/en/download/
- NodeJS installiert automatisch NPM mit

## Das erste Programm

- Nach der Node Installation können Javascript Programme mit `node` ausgeführt werden

```js
const message = "Hello World";
console.log(message);
```

```shell
node program.js
```

## Standalone NodeJS

- Mit dem Befehl `node` kann eine interaktive Javascript Umgebung aufgerufen werden
- Hier können beliebige Javascript Befehle ausgeführt werden

```shell
node
```

## Mit Modulen Arbeiten

- NodeJS kann mit CommonJS als auch ECMAscriot Modules (ESModules) verwendet werden
  - Als Standard wird das CommonJS Module System verwendet
  - Mit der Dateiendung `.mjs` können ESModules verwendet werden
  - Alternativ kann in der package.json der module type definiert werden

### CommonJS
- In CommonJS werden Funktionalitäten aus Node mit `require` importiert werden
```js
const fs = require('fs');
const data = fs.readFileSync('users.txt', 'utf8');
console.log(data.toString())
```

### ESModules
- In ESModules kann auf die `import` Syntax zurückgegriffen werden

```js
import { readFileSync } from 'node:js';

const data = fs.readFileSync('users.txt', 'utf8');
console.log(data.toString())
```

## NPM Packages installieren

- In Javascript Programmen ist es üblich mit externen paketen zu arbeiten
- Mit NPM können wir solche Pakete unserer Anwendung hinzufügen
- Pakete werden nur lokal installiert
- Um ein Projekt für NPM vorzubereiten, verwendet man den `npm init` befehl

```shell
npm init
```

- Nachdem man das Setup abgeschlossen hat liegt eine `package.json` Datei im Verzeichnis

```json
{
  "name": "my-project",
  "version": "1.0.0",
  "description": "My first NodeJS project",
  "main": "index.js",
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1",
  },
  "author": "Jens Reiner",
  "license": "ISC"
}
```

- Die `package.json` ist die wichtigste Datei in einem Node Projekt
- Sie beschreibt welche Pakete unsere Anwendung verwendet
- Sie kann auch Scripte ausführen wie z.B. `build` und `test`
- Ein Paket kann über `npm install <paket name>` installiert werden

```shell
npm install express
```

- In unserer `package.json` finden wir nun das express Paket unter `dependencies`

```json
{
  "name": "my-project",
  "version": "1.0.0",
  "description": "My first NodeJS project",
  "main": "index.js",
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1",
  },
  "author": "Jens Reiner",
  "license": "ISC",
  "dependencies": {
    "express": "^4.17.1"
  }
}
```

- Die Installation von express hat auch weitere Abhängigkeiten installiert
- NPM legt alle installierten `dependencies` in einen `node_modules` Ordner ab
- Um ein Projekt initial zu installieren kann `npm install` verwendet werden
- Innerhalb der `package.json` Datei kann die property `type: module` angegeben werden, um Node Programme als ESModule auszuführen

```json
{
  "name": "my-project",
  "version": "1.0.0",
  "description": "My first NodeJS project",
  "main": "index.js",
  "type": "module",
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1",
  },
  "author": "Jens Reiner",
  "license": "ISC",
  "dependencies": {
    "express": "^4.17.1"
  }
}
```

## Übung

- Installieren Sie sich ein Paket aus npm und integrieren sie die Funktionalität in ihrem Programm
  - bspw: date-fns
