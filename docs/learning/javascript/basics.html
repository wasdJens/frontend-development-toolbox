Grundlagen

Lernziele​

  • Was ist Javascript
  • Wie kann ich Variablen definieren
  • Was sind Objekte in Javascript
  • Was sind Datentypen in Javascript

Rückblick​

  • Javascript wurde 1995 eingeführt, um Programme für Webseiten zu schreiben
  • Mittlerweile ist Javascript Teil von jedem modernen Browser und ermöglicht die Interaktion mit Webseiten ohne diese neu-zuladen
  • Javascript hat nichts mit Java zu tun
  • Der Standard für Javascript wird als ECMAScript bezeichnet
    • Beschreibt wie Javascript funktioniert
    • Nicht jeder Browser supportet jedes Javascript Feature (vgl. Internet Explorer)

Javascript im Browser​

  • Jeder mit einem Browser kann Javascript schreiben
    • Öffnet ein neues Browser Fenster und öffnet die Entwickler Konsole (F12 oder Rechtsklick auf der Seite)

Variablen​

  • Variablen bilden die Grundlagen, um Daten zu speichern
    • Bspw. Möchten wir auf unserer Webseite Daten über den Benutzer speichern
  • In JS gibt es drei Möglichkeiten Variablen zu definieren
    • Eine Variable ist immer ein named storage für Daten
    • Variablen können beliebige Werte annehmen

Deklaration​

  • Die folgenden Statements deklarieren eine Variable in JS
let firstVariable;

const secondVariable;

var thirdVariable;
  • Historisch bedingt gibt es das var keyword noch.
    • Man sollte immer let und const verwenden.

Zuweisung von Werten​

  • Werte können mit = Variablen zugewiesen werden
let firstVariable;
firstVariable = "Hello";

// Alternative
let firstVariable = "Hello";

Konstanten​

  • Konstante definieren in Javascript
    • Keyword const
    • Konstante Variablen können nicht überschrieben werden
const myConstant = "Hello";
myConstant = "Changed" // Error

Datentypen​

  • Ein Wert in Javascript hat immer einen Datentyp
  • In JavaScript gibt es 8 Datentypen (Symbol betrachten wir hier nicht)
  • Javascript ist dynamisch typisiert
    • Das bedeutet einer Variable ist kein explizierter Datentyp zugeordnet
    • Ein Datentyp von einer Variable kann überschrieben werden
let input = "Foo";
input = 1337;

Datentypen: Number​

  • In JavaScript können Numbers Integer und Float Werte annehmen
  • Zusätzlich gibt es noch BigInt (Zahlen > 2^53−1)
let number = 1337;
number = 42.42;

const largeNumber = BigInt(9007199254740991)

Datentypen: String​

  • Strings in JavaScruot können entweder mit " oder ' deklariert werden
  • String Verkettung mittels +
const aString = "Hello World";
const bString = 'Hello World'';

const h = "Hello";
const w = "World";
console.log(h + " " + w);
  • Template Strings sind eine besondere Art strings zu definieren
  • Mit einem backtick ``` können Strings definieren werden, welche gleichzeitig Funktionen ausführen können
  • So lassen sich Variablen direkt in einem String ausgeben
const number = 800;
const price = `The price of the new phone is ${number}`;
console.log(price); // The price of the new phone is 800
const phone = 800;
const discount = 25;
const price = `The price of the new phone is ${phone - ((phone / 100) * discount)}`;
console.log(price); // The price of the new phone is 600 
const phone = 800;
const discount = 25;
const price = `The price of the new phone is ${phone} with a discount of ${discount}`;
console.log(price);

Datentypen: Boolean​

  • Der Logische Datentyp welcher nur zwei Zustände annehmen kann 
const confirmed = false;
const approved = true;

Datentypen: null & undefined​

  • In Javascript gibt es zwei spezielle Datentypen null und undefined
  • null repräsentiert nothing, empty oder wert unbekannt
  • undefined hingegen bedeutet, dass einer Variable noch kein Wert zugewiesen wurde
const username = null;

const email;
console.log(email) // undefined
  • null kann also speziell verwendet werden, um eine Variable als leer oder unbekannt zu deklarieren
  • undefined hingegen symbolisiert das eine Variable keinen Wert hat

Datentypen: Objekte​

  • Objekte in JavaScript sind der wichtigste Datentyp
  • Objekte können verschiedene Properties beinhalten
  • Jedes Property hat einen Wert
  • Das Property ist dann der Schlüssel, um auf den Wert zuzugreifen
const user = {
  name: "Jens",
  age: 26,
  gender: "m",
  isAdmin: true
}
  • Objekte können mit {} definiert werden
  • Objekte können Variablen zugeordnert werden
const phone = {
  manufacturer: "Apple",
  model: "iPhone",
  storage: 128,
  isReleased: false
}
  • Objekte können auch leer definiert werden
  • Objekte haben einen key und einen dazugehörigen wert { <key> : <value> }
  • Properties können wir einfach auslesen in einem Objekt (-> . notation)
console.log(user.name) // => Jens
  • Properties können jeden Datentyp beinhalten
  • Neue Werte können mit der . Notation auch hinzugefügt werden
let machine = {
  serialNo: 1234,
  name: "Fast Machine"
}
machine.price = 2500;
console.log(machine); // Object { serialNo: 1234, name: "Fast Machine", price: 2500}
  • Properties entfernt man mit dem delete keyword
delete machine.price
console.log(machine); // Object { serialNo: 1234, name: "Fast Machine" }
  • Properties verändern mit der . Notation
machine.serialNo = 42;
console.log(machine); // Object { serialNo: 42, name: "Fast Machine" }
  • Der key von einer property kann auch ein multi-word sein
const factory = {
  "construction site": "Heidenheim an der Brenz"
};
console.log(factory);
  • Die . Notiation funktioniert in diesem Fall nicht mehr
  • Um auf eine multi-word property zuzugreifen verwendet man [ ] (Square bracket notiation)
console.log(factory["construction site"]); // Heidenheim an der Brenz
  • Die square bracket notiation funktioniert auch für Variablen 
const key = "consturction site"
console.log(factory[key]); // Heidenheim an der Brenz
  • Die square bracket notation kann auch keys in properties definieren (Computed properties)
const fruit = "apple";
const bag = {
  [fruit]: 5
}
console.log(bag) // Object { apple: 5}

Datentypen: Typeof operator​

  • Um in Javascript herauszufinden was für ein Datentypen eine Variable ist gibt es eine speziellen Operator typeof
const price = 25;
console.log(typeof price); // number
console.log(typeof "Foo"); // string
console.log(typeof null); // object

Recap​

  • Javascript hat 8 unterschiedliche Datentypen
  • Javascript ist dynamisch typisiert d.h. eine Variable kann mehrere Datentypen zur Laufzeit annehmen
  • Der wichtigste Datentyp ist Object, um komplexe Datenstrukturen abzubilden
  • undefined und null sind spezielle Datentypen
    • null sollte immer als „leerer“ Wert gesehen werden
    • undefined ist eine variable ohne wert

  • Definieren Sie sich 10 unterschiedliche Variablen
  • Definieiren Sie sich zwei Objekte
  • Verändern Sie die Werter ihrer Variablen
